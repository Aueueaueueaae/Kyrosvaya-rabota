# Облачное файловое хранилище с клиентским шифрованием

Два контейнера под `docker-compose`:
1) `server` (FastAPI) хранит только зашифрованные данные и метаданные шифрования  
2) `client` (Streamlit) выполняет шифрование/дешифрование на стороне пользователя

## Что поддерживается

- Загрузка файла на сервер после клиентского шифрования (AES-256-GCM)
- Список файлов
- Скачивание и расшифровка по паролю
- Удаление файлов

## Как работает шифрование

- Ключ AES-256-GCM выводится из пароля через PBKDF2-HMAC-SHA256
- Для каждого файла генерируются:
  - `salt` (16 байт) для PBKDF2
  - `nonce` (12 байт) для AES-GCM
- Сервер не знает пароль и не умеет расшифровывать данные

## Требования

- Docker Desktop
- WSL (для винды)

## Запуск

В папке проекта:

```bash
docker compose up --build
```

Далее, после сборки и поднятия контейнеров:
- Streamlit UI: `http://localhost:8501`

## Использование

1. Введите `Server URL` (по умолчанию уже выставлен: `http://server:8000` для контейнеров).
2. Задайте пароль шифрования.
3. Загрузите файл кнопкой `Encrypt & Upload`.
4. Нажмите `Refresh list` чтобы увидеть файлы.
5. Выберите файл и нажмите `Decrypt & Download`, чтобы скачать расшифрованную копию.
6. При необходимости удалите файл кнопкой `Delete`.

## Хранилище данных

Зашифрованные файлы сохраняются в контейнере `server` в volume `server_storage` по пути `/data/storage`.
Пароли никуда не отправляются и не сохраняются.

## Структура API

- `POST /api/upload`
- `GET /api/files`
- `GET /api/files/{id}`
- `DELETE /api/files/{id}`
